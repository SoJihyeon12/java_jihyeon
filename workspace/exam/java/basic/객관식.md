# 자바 프로그래밍 기초 객관식 문제 10선

## 문제 1
컴파일러 언어와 인터프리터 언어에 대한 설명으로 올바르지 않은 것은?
1) 컴파일러 언어는 전체 소스 코드를 한 번에 기계어로 변환하여 실행 파일을 만든다.
2) 인터프리터 언어는 코드를 실행할 때 한 줄씩 즉석에서 번역하여 실행하므로 개발 속도가 빠른 편이다.
3) 자바는 소스 코드를 바이트코드로 컴파일한 뒤 JVM을 통해 실행하는 하이브리드 언어이다.
4) 컴파일러 언어로 생성된 기계어 실행 파일은 어떤 운영체제(OS)에서든 수정 없이 그대로 실행할 수 있다.

정답: 4
해설: 컴파일러 언어로 빌드된 결과물은 특정 CPU 아키텍처 및 OS에 종속적이므로 플랫폼 독립적이지 않습니다. 반면 자바는 JVM을 사용해 플랫폼 독립성을 제공합니다.

---

## 문제 2
JDK와 JRE에 대한 설명으로 올바르지 않은 것은?
1) JRE는 JVM과 자바 프로그램 실행에 필요한 핵심 라이브러리를 포함한다.
2) JDK는 JRE를 포함하며, 자바 컴파일러(javac) 등 개발에 필요한 도구들을 추가로 포함한다.
3) javac는 자바 소스 파일(.java)을 바이트코드(.class)로 변환하는 컴파일러이다.
4) java 명령은 소스 코드를 직접 컴파일하고 곧바로 실행까지 마치는 도구이다.

정답: 4
해설: java 명령은 이미 컴파일된 바이트코드(.class)를 실행하는 인터프리터로, 내부의 JVM을 구동합니다. 소스 코드를 직접 컴파일하는 도구는 javac입니다.

## 문제 3
자바의 참조 데이터 타입(Reference Type)에 대한 설명으로 올바르지 않은 것은?
1) 기본 데이터 타입(Primitive Type) 8개를 제외한 모든 데이터 타입은 참조 데이터 타입이다.
2) 참조 타입 변수는 객체가 메모리 상에 실제 올라가 있는 위치 주소(Reference)를 가리킨다.
3) 참조 타입 변수의 크기는 저장되는 객체의 크기에 비례하여 동적으로 변한다.
4) 참조 변수가 아무런 객체도 가리키지 않는 상태를 표현하기 위해 'null' 값을 사용할 수 있다.

정답: 3
해설: 참조 타입 변수는 실제 객체가 아닌 객체의 주소값(주소 크기는 시스템 아키텍처에 따라 고정됨)을 가지므로, 객체의 크기와 상관없이 참조 변수 자체의 메모리 크기는 고정되어 있습니다.

## 문제 4
자바의 식별자(Identifier) 명명 규칙 및 예약어에 대한 설명으로 올바르지 않은 것은?
1) 식별자 이름에 달러 기호('$')와 밑줄('_')을 사용할 수 있다.
2) 식별자는 숫자로 시작할 수 없다.
3) 자바의 키워드인 예약어('int', 'class', 'public' 등)도 변수명 등의 식별자로 사용할 수 있다.
4) 클래스 이름은 대문자로 시작하는 PascalCase를 사용하는 것이 개발자 간의 관례이다.

정답: 3
해설: 자바 언어 문법에서 미리 약속되어 특별한 의미로 사용되는 예약어(Keyword)는 식별자로 사용할 수 없습니다.

## 문제 5
다음 중 자바에서 컴파일 에러 없이 자동 형변환(묵시적 형변환)이 정상적으로 수행되는 코드는 무엇인가?
1) int a = 10; byte b = a;
2) double pi = 3.14; float f = pi;
3) int num = 100; double d = num;
4) long big = 3000000000L; int i = big;

정답: 3
해설: 자동 형변환은 더 작은 크기의 데이터 타입에서 더 큰 크기의 데이터 타입으로 대입할 때 발생합니다. int(4바이트)는 double(8바이트)보다 작으므로 3번은 자동 형변환이 됩니다. 1, 2, 4번은 강제 형변환(명시적 형변환)이 필요하며 그렇지 않으면 컴파일 에러가 발생합니다.

## 문제 6
다음 삼항 연산자가 포함된 자바 코드의 실행 결과로 올바른 것은?
```java
int score = 85;
String grade = (score >= 90) ? "훌륭해요" : (score >= 80) ? "잘했어요" : "분발하세요";
System.out.println(grade);
```
1) 훌륭해요
2) 잘했어요
3) 분발하세요
4) 85

정답: 2
해설: score가 85이므로 첫 번째 조건 'score >= 90'은 거짓이 됩니다. 이에 따라 거짓 항에 해당하는 두 번째 삼항 연산식인 '(score >= 80) ? "잘했어요" : "분발하세요"'가 수행되는데, 'score >= 80'은 참이므로 최종적으로 "잘했어요"가 반환되어 grade에 저장됩니다.

## 문제 7
자바의 반복문인 'for'문과 'while'문에 대한 설명으로 가장 올바르지 않은 것은?
1) 'for'문은 주로 반복 횟수가 명확하게 정해져 있을 때 사용하기에 적합하다.
2) 'while'문은 주로 반복 횟수는 정해져 있지 않지만 특정 조건이 충족되는 동안 루프를 유지할 때 적합하다.
3) 'while'문은 조건식이 처음부터 'false'이면 단 한 번도 실행되지 않고 루프를 건너뛴다.
4) 'for'문의 초기화식, 조건식, 증감식은 생략이 불가능하며 생략 시 컴파일 에러가 발생한다.

정답: 4
해설: 'for'문의 초기화식, 조건식, 증감식은 필요에 따라 생략할 수 있습니다. 예를 들어 'for(;;)'와 같이 모두 생략하면 무한 루프가 됩니다.

## 문제 8
자바의 향상된 'for'문(Enhanced 'for' loop)에 대한 설명으로 올바르지 않은 것은?
1) 배열이나 컬렉션의 모든 요소를 인덱스 없이 처음부터 끝까지 순차적으로 탐색할 때 사용한다.
2) 일반 'for'문에 비해 코드가 직관적이고 인덱스 범위를 초과하는 에러를 방지할 수 있다.
3) 루프 내부에서 탐색 중인 배열 요소의 값을 직접 변경하여 원본 배열을 수정할 수 있다.
4) 값을 읽기만 하는 읽기 전용 탐색 작업에 적합하다.

정답: 3
해설: 향상된 'for'문은 배열의 요소를 임시 변수에 담아 사용하는 방식이므로, 루프 내에서 변수 값을 변경해도 원본 배열의 실제 요소 값은 변경되지 않습니다.

## 문제 9
자바의 스택(Stack) 영역과 힙(Heap) 영역에 대한 설명으로 올바르지 않은 것은?
1) 기본 데이터 타입(Primitive Type)의 지역 변수는 스택 영역에 실제 데이터 값을 직접 저장한다.
2) 'new' 연산자로 생성된 배열과 인스턴스(객체)는 힙 영역에 생성된다.
3) 참조 데이터 타입(Reference Type) 변수는 힙 영역에 생성된 실제 배열이나 인스턴스(객체)의 주소값을 저장한다.
4) 힙 영역에 생성된 배열과 인스턴스(객체)는 스택 영역에서 자신을 참조하는 변수가 추가되면 메모리에 새로 복사되어 할당된다.

정답: 4
해설: 힙 영역에 생성된 객체를 다른 참조 변수에 대입하면 객체가 복사되는 것이 아니라, 객체의 주소값만 복사됩니다(얕은 복사). 더 이상 참조되지 않는 객체는 가비지 컬렉터(GC)에 의해 자동으로 수거됩니다.

---

## 문제 10
자바의 메서드(Method) 정의 및 호출에 대한 설명으로 올바르지 않은 것은?
1) 메서드는 특정 기능을 수행하는 코드들을 하나의 블록으로 묶어 재사용성을 높인 것이다.
2) 메서드 선언부에 정의되어 외부에서 전달되는 값을 받는 변수를 '매개변수(Parameter)'라고 한다.
3) 메서드를 호출할 때 실제로 전달하는 구체적인 값을 '인자(Argument)'라고 한다.
4) 메서드의 반환 타입이 'void'인 경우에도 반드시 'return' 키워드 뒤에 반환할 값을 적어야 한다.

정답: 4
해설: 반환 타입이 'void'인 메서드는 반환할 값이 없음을 의미하므로 'return' 키워드를 생략하거나 값 없이 'return;' 형태로만 사용해야 합니다.
